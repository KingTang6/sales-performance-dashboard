# Data Dictionary

This data dictionary documents the cleaned dataset used in the Sales Performance Analytics Dashboard project. The cleaned file is stored at `data/superstore_cleaned.csv`.

## Cleaned Dataset Fields

| Field | Type | Description | Used For |
| --- | --- | --- | --- |
| `RowID` | Integer | Unique row-level identifier from the source dataset. | Row tracking and data validation. |
| `OrderID` | Integer | Unique order identifier. Multiple rows can belong to the same order. | Distinct order counts and order-level analysis. |
| `OrderDate` | Date | Date when the order was placed, standardized as `yyyy-MM-dd`. | Time-based sales and profit analysis. |
| `ShipDate` | Date | Date when the order was shipped, standardized as `yyyy-MM-dd`. | Potential shipping and fulfillment analysis. |
| `Year` | Integer | Calendar year extracted from `OrderDate`. | Year-level filtering and grouping. |
| `Month` | Integer | Calendar month extracted from `OrderDate`. | Monthly trend analysis. |
| `YearMonth` | Text | Combined year-month value from `OrderDate`, formatted as `yyyy-MM`. | Monthly sales and profit trend reporting. |
| `Region` | Text | Sales region associated with the order. | Regional sales, profit, and margin comparison. |
| `Province` | Text | Province associated with the customer/order. | More detailed geographic analysis. |
| `CustomerSegment` | Text | Customer type used to compare contribution across segments. | Customer contribution and segment performance analysis. |
| `Category` | Text | Main product category. | Product category profitability analysis. |
| `SubCategory` | Text | More detailed product sub-category. | Product-level margin risk and profitability analysis. |
| `ProductName` | Text | Product name from the transaction row. | Product-level review if deeper analysis is needed. |
| `Sales` | Decimal | Revenue generated by the transaction row. | Total sales, trend analysis, region/category comparisons. |
| `Profit` | Decimal | Profit generated by the transaction row. | Total profit, profit margin, and low-profit diagnostics. |
| `Quantity` | Integer | Number of units sold on the transaction row. | Sales volume and quantity contribution analysis. |
| `Discount` | Decimal | Discount rate applied to the transaction row. | Discount and margin review. |
| `UnitPrice` | Decimal | Unit price for the product. | Pricing context and product analysis. |
| `ShippingCost` | Decimal | Shipping cost associated with the transaction row. | Profit quality and cost driver review. |
| `ProfitMargin` | Decimal | Row-level margin calculated as `Profit / Sales`. | Margin diagnostics and low-profit identification. |

## Derived Fields

The following fields were created during cleaning:

- `Year`: extracted from `OrderDate`
- `Month`: extracted from `OrderDate`
- `YearMonth`: created from `OrderDate` for monthly trend analysis
- `ProfitMargin`: calculated as `Profit / Sales`

## Notes

- `OrderID` is not unique at the row level because one order can contain multiple products.
- `ProfitMargin` is calculated at the row level in the cleaned dataset. Summary-level margin is calculated as total profit divided by total sales.
- The dataset does not include marketing spend, inventory, customer acquisition cost, or return data.
